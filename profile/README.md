# yodo-test-org

## About Us

yodo-test-org is a GitHub organization used to organize repositories, shared engineering practices, and security policies.

We keep this organization intentionally small and explicit: repositories should have a clear owner, a clear purpose, and enough documentation for another maintainer to operate them safely.

## What We Maintain

- Repositories for experiments, validation, and shared tooling
- Organization-wide GitHub health files
- Security and contribution policies that apply when a repository does not define its own

## Repository Standards

Repositories in this organization should include:

- A README that explains the purpose, setup, and basic usage
- A license when the repository is intended to be reused or shared publicly
- Clear ownership or maintainer information
- Security reporting guidance, either repository-specific or inherited from this organization profile

## Contributing

Before opening an issue or pull request, check the target repository README. Repository-specific instructions override the organization defaults in this  repository.

## Security

Please do not report security issues in public issues. Use the guidance in [SECURITY.md](https://github.com/yodo-test-org/.github/blob/main/SECURITY.md).
